# 生成文档 (CEP/CER/DCR)

## 概述

当临床评价工作流的所有步骤完成并批准后，可一键生成三份合规文档。

## 文档类型

| 文档 | 全称 | 生成方式 |
|------|------|---------|
| CEP | Clinical Evaluation Plan (临床评价计划) | AI 生成 + 代码注入 |
| CER | Clinical Evaluation Report (临床评价报告) | AI 生成 + 代码注入 |
| DCR | Data Collection Report (数据收集报告) | 纯代码确定性组装 |

### CEP (临床评价计划)

涵盖临床评价的范围、方法、数据来源等规划内容。文档结构遵循 MEDDEV 2.7/1 Rev.4 指南。

### CER (临床评价报告)

完整的临床评价报告，包含所有分析结果、获益-风险评估和结论。是提交公告机构审核的核心文档。

### DCR (数据收集报告)

系统性的数据收集记录，包含检索策略、文献统计、筛选结果、评分表格等客观数据。全部由代码确定性组装，确保数据准确性。

## 生成操作

### Document Pipeline (文档流水线)

在项目仪表板中可看到「Document Pipeline / 文档流水线」区域：

1. 每份文档显示当前状态 (Ready / Generating / Completed)
2. 点击「Generate / 生成」启动生成
3. 生成过程为流式输出，可实时查看进度
4. 生成完成后状态变为 Completed

### 文档操作

生成完成后可执行：

| 操作 | 说明 |
|------|------|
| Insert to Word | 将文档内容插入到当前 Word 文档 |
| Download | 下载为 Word (.docx) 文件 |
| Preview | 预览文档内容 |
| Edit | 编辑文档文本 |
| Redo | 重新生成（需确认，将覆盖当前内容） |

## 文档格式

生成的文档遵循统一的排版规范：

- 字体：Times New Roman
- 正文：10.5pt，1.5 倍行距，两端对齐
- 标题层级：h1(16pt) / h2(15pt) / h3(14pt)
- 表格：1pt 实线边框，9pt 字体，灰色表头
- 页眉/页脚：包含文档编号和页码

## AI 内容标注

根据合规要求，所有 AI 生成的文档：
- 封面标注「AI-Assisted Document -- Generated with Reguverse Assistant」
- Word 文档属性中嵌入 AI 生成元数据

## 注意事项

- 文档生成可能需要 1-3 分钟（取决于内容长度）
- 如果生成中断，可重试
- 生成的文档内容应人工审阅后方可提交监管机构
- 建议在生成前确保所有步骤的内容准确无误
