# MDCG 2019-16 rev.1：医疗器械网络安全指南

**文件编号**：MDCG 2019-16 rev.1  
**发布日期**：2020年12月（rev.1）  
**适用法规**：EU MDR 2017/745 / EU IVDR 2017/746  
**官方PDF**：[下载](https://health.ec.europa.eu/document/download/b23b362f-8a56-434c-922a-5b3ca4d0a7a1_en?filename=md_cybersecurity_en.pdf)

## 文件目的

本指南为医疗器械（包括医疗器械软件MDSW）的网络安全要求提供实践指导，帮助制造商满足EU MDR/IVDR中关于网络安全的GSPR要求。

## 网络安全的法规基础

### GSPR相关要求

EU MDR附件I（GSPR）中与网络安全相关的要求：
- **第17.2节**：含有电子可编程系统的器械须设计为确保可重复性、可靠性和性能
- **第17.4节**：须保护免受未经授权访问
- 网络安全须纳入风险管理（EN ISO 14971）

### 适用范围

本指南适用于：
- 医疗器械软件（MDSW）
- 含有软件的医疗器械
- 连接网络的医疗器械（包括IoMT）

## 网络安全生命周期

### 安全开发生命周期（SDLC）

制造商须将网络安全整合到产品开发全生命周期：

**设计阶段**：
- 安全架构设计
- 威胁建模（STRIDE方法）
- 安全需求定义

**开发阶段**：
- 安全编码规范
- 代码审查
- 静态代码分析

**验证阶段**：
- 渗透测试
- 漏洞扫描
- 安全功能测试

**上市后阶段**：
- 漏洞监测
- 安全补丁管理
- 事件响应计划

## 技术文件要求

### 须包含的网络安全文件

| 文件类型 | 内容要求 |
|---------|---------|
| **安全架构文档** | 系统架构、数据流图、信任边界 |
| **威胁建模报告** | 识别的威胁、攻击面分析 |
| **风险评估** | 网络安全风险与安全措施 |
| **安全测试报告** | 渗透测试、漏洞扫描结果 |
| **上市后网络安全计划** | 漏洞监测、补丁管理流程 |

### 最低安全要求

**认证与授权**：
- 强密码策略
- 多因素认证（高风险器械）
- 基于角色的访问控制

**数据保护**：
- 传输加密（TLS 1.2+）
- 存储加密（敏感数据）
- 数据完整性验证

**审计日志**：
- 记录安全相关事件
- 日志防篡改保护
- 日志保留策略

**软件更新**：
- 安全的软件更新机制
- 更新完整性验证
- 回滚能力
